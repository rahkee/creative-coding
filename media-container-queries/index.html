ðŸŒ³ DOM Tree Architecture

Media queries live towards the top of the DOM tree for site-wide changes. There shouldn't be many media queries.

Container queries live in the lower branches for localized components. It's okay to have many container queries.

ðŸ“± Media Queries

What: Respond to viewport (browser window) size

Best for: Overall page layout changes like sidebar to hamburger menu conversions, global typography changes, and device-specific features (orientation, hover capability)

Pros:

  • Great for global layout changes
  • Can query many device features

Cons:

  • Elements can overflow or overlap before the viewport breakpoint triggers
  • Requires mental math to predict how components will behave at different viewport sizes

ðŸ“¦ Container Queries

What: Respond to parent container size

Best for: Component-based responsive design

Pros:

  • More intuitive - no mental math required
  • Based on localized dimensions rather than global viewport
  • Works anywhere in the layout

Cons:

  • Requires container-type declaration
